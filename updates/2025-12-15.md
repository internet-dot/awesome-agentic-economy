## 1. FDIC Proposes Stablecoin Issuance Process Under GENIUS Act

**Published**: December 16, 2025
**Status**: Notice of Proposed Rulemaking (NPRM)

### Overview

The Federal Deposit Insurance Corporation (FDIC) Board of Directors approved a notice of proposed rulemaking that would implement application provisions under the Guiding and Establishing National Innovation for U.S. Stablecoins Act (GENIUS Act). This is not a routine bureaucratic update—it's the "firing gun" for the banking sector's entry into the agentic economy.

The GENIUS Act, signed into law on July 18, 2025, established the first comprehensive federal framework for digital assets in the United States. The December 16 NPRM clarifies a crucial architectural decision: banks cannot issue stablecoins directly from their insured balance sheets but must form a **Permitted Payment Stablecoin Issuer (PPSI)** subsidiary.

### Technical Details

Key provisions from the FDIC proposal:

- **PPSI Subsidiary Structure**: Banks must create separate subsidiaries to issue payment stablecoins
- **Risk Isolation**: Structure protects the Deposit Insurance Fund (DIF) if an agent swarm triggers a "run"
- **Operational Separation**: Strict separation of governance, capital, and systems required
- **Safe Harbor Provision**: Temporary waiver for applicants pending as of the Act's effective date
- **Comment Period**: Comments due by February 17, 2026

### Developer Integration Example

Conceptual pattern from FDIC application frameworks:

```python
import requests

def genius_act_application(issuer_data):
    """
    Conceptual implementation for GENIUS Act stablecoin
    issuance application process.
    """
    payload = {
        'issuer': issuer_data,
        'compliance': 'genius-act',
        'subsidiaryType': 'PPSI',
        'stablecoin': {
            'type': 'payment-stablecoin',
            'backing': 'treasury-bills',
            'currency': 'USD'
        },
        'riskIsolation': {
            'separateGovernance': True,
            'separateCapital': True,
            'difProtection': True
        }
    }

    response = requests.post(
        'https://api.fdic.gov/stablecoin/apply',
        json=payload,
        headers={'Authorization': 'Bearer API_KEY'}
    )

    if response.ok:
        return response.json()['applicationStatus']
    return {'error': 'Application failed', 'details': response.text}

# Usage: genius_act_application({'entity': 'bank', 'reserves': 1e9, 'fdic_supervised': True})
```

## 5. Brookings on Stablecoins Issues for GENIUS Act Implementation

**Published**: October 2025 (Updated December Context)
**Status**: Policy Analysis

### Overview

Brookings Institution highlighted four key issues for regulators implementing stablecoins under the GENIUS Act. The analysis identifies reserve management, interoperability, fraud prevention, and systemic risk as critical implementation challenges that the December FDIC proposal aims to address.

The Brookings perspective emphasizes that the PPSI subsidiary structure is designed specifically to address systemic risk concerns while enabling innovation.

### Technical Details

Four key implementation issues:

- **Reserve Requirements**: 1:1 backing in high-quality liquid assets (HQLA) like T-bills
- **Risk Management**: Frameworks for addressing stablecoin run scenarios
- **Interoperability**: Cross-platform and cross-chain compatibility requirements
- **Fraud Prevention**: Authentication and identity verification for agentic use cases

### Developer Integration Example

Conceptual pattern for GENIUS Act issue validation:

```python
def genius_issues_check(issuer):
    """
    Validate issuer against four key Brookings-identified
    GENIUS Act implementation concerns.
    """
    issues = {
        'reserve': validate_reserve_backing(issuer),
        'risk': validate_risk_management(issuer),
        'interop': validate_interoperability(issuer),
        'fraud': validate_fraud_prevention(issuer)
    }

    for issue, result in issues.items():
        if not result.passed:
            return {
                'status': 'Issue detected',
                'issue': issue,
                'details': result.reason
            }

    return {'status': 'Compliant', 'issuer': issuer.id}

# Usage: genius_issues_check({'type': 'stablecoin-issuer', 'reserves': 1e9})
```

## 7. Hong Kong Stablecoins Framework Context

**Published**: December 2024 (Gazettal) / August 2025 (Effective)
**Status**: Active Supervision Phase

### Overview

Hong Kong's stablecoin regulatory framework provides context for the global regulatory convergence happening in December 2025. The regime, first gazetted in December 2024 and effective since August 2025, is now in full supervision phase with the HKMA actively consulting on custody and OTC requirements.

Hong Kong serves as the "counter-factual" to the US GENIUS Act: implementation happened faster (August 2025 vs. expected mid-2026 for GENIUS Act final rules), but the US framework's banking integration may provide greater liquidity for agentic applications.

### Technical Details

Hong Kong framework characteristics:

- **Fiat-Referenced Focus**: Strictly focused on fiat-backed stablecoins
- **Faster Implementation**: Active supervision since August 2025
- **HKMA Oversight**: Central bank supervision with custody requirements
- **Asia-Pacific Gateway**: Regional hub for compliant stablecoin operations
