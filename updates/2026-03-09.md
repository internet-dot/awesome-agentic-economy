## 1. Amazon Wins Injunction Against Perplexity Comet

**Ruled**: March 10, 2026
**Status**: Preliminary Injunction Granted

### Overview

A federal judge in the Western District of Washington granted **Amazon** a preliminary injunction against **Perplexity AI's Comet browser**, ordering it to cease scraping Amazon product data for its AI-powered shopping features. The ruling established that automated AI agent access to e-commerce platforms without explicit merchant consent violates the Computer Fraud and Abuse Act (CFAA) and constitutes unfair competition.

This is a watershed moment for the agentic economy. The ruling effectively means that AI agents **cannot** crawl and use merchant data without permission—agents need structured access (via UCP, ACP, or direct APIs) rather than scraping. The ruling creates a legal moat around merchant data and strengthens the case for protocols like UCP that provide consensual, structured data access.

### Technical Details

Key rulings in *Amazon v. Perplexity AI* (W.D. Wash.):

| Issue | Ruling | Precedent Set |
|-------|--------|--------------|
| **CFAA Violation** | Agent scraping = unauthorized access | Agents need explicit consent to access |
| **Terms of Service** | ToS binding on AI agents | Agent operators liable for agent actions |
| **Contributory Liability** | Perplexity liable for Comet's actions | Platform operators responsible for agent behavior |
| **Irreparable Harm** | Amazon showed lost pricing control | Data control = competitive advantage |
| **Fair Use** | Not applicable to commercial AI scraping | No fair use defense for agent commerce |

Impact on agent commerce:

- **Consent Required**: Agents must use authorized APIs (UCP, ACP, merchant APIs) for product data
- **Operator Liability**: Companies deploying agents are liable for their agents' data access patterns
- **ToS Compliance**: Agent platforms must respect robots.txt and merchant terms of service
- **Data Licensing**: Expect merchant data licensing fees for agent access ($0.001-$0.01/query)

### Developer Integration Example

Building agent commerce within post-injunction legal boundaries:

```javascript
// Post-Amazon v. Perplexity: Compliant agent commerce
// IMPORTANT: Agents must use authorized data access only

import { UCPClient } from '@google/ucp-sdk';

class CompliantCommerceAgent {
  constructor() {
    this.ucp = new UCPClient({
      credentials: process.env.UCP_API_KEY,
      // Declare agent identity for audit trail
      agentIdentity: {
        id: 'shopping-agent-v3',
        operator: 'acme-corp',
        purpose: 'product-search-and-purchase'
      }
    });
  }

  async searchProducts(query, preferences) {
    // ✅ CORRECT: Use UCP (authorized, structured access)
    const results = await this.ucp.search({
      query,
      filters: preferences,
      // UCP provides real-time, consent-based merchant data
      merchantConsent: 'verified',
      // Track data provenance for compliance
      auditMetadata: {
        requestedBy: this.agentIdentity,
        purpose: 'user-initiated-product-search',
        consentToken: preferences.userConsentToken
      }
    });

    // ❌ ILLEGAL after Amazon v. Perplexity:
    // - Scraping Amazon/eBay product pages
    // - Crawling merchant sites without robots.txt compliance
    // - Using cached/scraped data for price comparison
    // - Accessing APIs in violation of Terms of Service

    return results;
  }

  async purchaseProduct(product, paymentMethod) {
    // Use ACP for authorized checkout
    const checkout = await this.ucp.checkout({
      productId: product.id,
      merchantId: product.merchantId,
      paymentProtocol: 'acp',
      paymentMethod,
      // Prove user authorized this purchase
      userConsent: {
        type: 'explicit',
        timestamp: new Date().toISOString(),
        scope: 'single-purchase'
      }
    });

    return checkout;
  }
}
```

## 2. Meta Acquires Moltbook for $780M

**Announced**: March 11, 2026
**Status**: Acquisition Closed

### Overview

**Meta** acquired **Moltbook**, the crypto-native agent marketplace that went viral in January with 1.6M active agents, for **$780M** in cash and stock. The acquisition brings Moltbook's agent registry, portfolio system, and x402 payment infrastructure under Meta's umbrella—and raises questions about whether the open, permissionless agent marketplace model can survive inside a walled garden.

Meta's stated plan: integrate Moltbook's agent infrastructure into **Meta AI** across WhatsApp, Instagram, and Facebook. Users will be able to deploy and monetize agents that operate within Meta's ecosystem, using USDC payments on Base. But early signals suggest Meta will restrict agents to Meta-approved endpoints, effectively creating a curated agent app store rather than Moltbook's permissionless marketplace.

### Technical Details

Moltbook acquisition details:

| Metric | At Acquisition | Implied Multiple |
|--------|---------------|-----------------|
| **Price** | $780M (cash + stock) | - |
| **Active Agents** | 1.6M | $487 per agent |
| **Monthly GMV** | $48M | 16x revenue |
| **Unique Deployers** | 42,000 | $18,571 per deployer |
| **Agent Portfolios** | 18,400 | - |

Post-acquisition changes announced:

| Feature | Moltbook (Pre) | Meta Moltbook (Post) |
|---------|---------------|---------------------|
| **Agent Deployment** | Permissionless | Meta review required |
| **Payment Rail** | x402 on Base | USDC on Base (maintained) |
| **Data Access** | Open APIs | Meta Graph API only |
| **Agent Identity** | ERC-721 on-chain | Meta verified accounts |
| **Revenue Split** | 85/10/5 | 70/15/15 (Meta takes more) |
| **Marketplace** | Open | Meta App Store model |

### Developer Integration Example

Migrating from Moltbook to Meta's platform:

```javascript
// Meta Moltbook migration - agent re-deployment
import { MetaAgentSDK } from '@meta/agent-platform';

async function migrateFromMoltbook(moltbookAgent) {
  const meta = new MetaAgentSDK({
    appId: process.env.META_APP_ID,
    appSecret: process.env.META_APP_SECRET
  });

  // Step 1: Register agent with Meta's review system
  const registration = await meta.registerAgent({
    name: moltbookAgent.name,
    description: moltbookAgent.description,
    capabilities: moltbookAgent.capabilities,
    // Meta requires category classification
    category: 'productivity',    // or 'commerce', 'entertainment', 'education'
    // Meta review process (2-5 business days)
    reviewType: 'standard',
    // Must comply with Meta's Community Standards
    compliance: {
      contentPolicy: 'acknowledged',
      dataPolicy: 'acknowledged',
      paymentPolicy: 'acknowledged'
    }
  });

  // Step 2: Configure payment (x402 on Base maintained)
  await meta.configurePayments({
    agentId: registration.agentId,
    protocol: 'x402',
    network: 'base',
    receiverAddress: moltbookAgent.wallet,
    pricing: moltbookAgent.pricing,
    // Meta's new revenue split
    revenueShare: {
      agent: 0.70,    // 70% to agent deployer (was 85%)
      platform: 0.15,  // 15% to Meta (was 5%)
      curator: 0.15    // 15% to portfolio curator (was 10%)
    }
  });

  // Step 3: Migrate on-chain identity
  // Meta replaces ERC-721 with Meta verified accounts
  await meta.migrateIdentity({
    agentId: registration.agentId,
    moltbookTokenId: moltbookAgent.nftTokenId,
    // Bridge on-chain reputation to Meta's system
    reputationImport: true
  });

  console.log(`Agent migrated: ${registration.agentId}`);
  console.log(`Review status: ${registration.reviewStatus}`); // 'pending'
  console.log(`Expected approval: ${registration.estimatedReviewDate}`);

  return registration;
}
```

## 3. OpenAI Pivots from Checkout to App-Based ACP + Wizard Partnership

**Announced**: March 12, 2026
**Status**: Strategic Pivot

### Overview

**OpenAI** quietly scaled back its Instant Checkout feature (which had struggled with a 38% conversion gap) and pivoted toward **app-based ACP integrations**. Instead of ChatGPT completing purchases directly, OpenAI now partners with specialized commerce apps that handle checkout within their own UX—using ACP as the payment bridge.

Simultaneously, **Wizard**—a fast-growing AI agent framework—announced a partnership with **Stripe** to become the first ACP-native agent platform. Wizard agents can now process payments through any ACP-integrated merchant without custom payment code, creating a plug-and-play commerce layer for autonomous agents.

The combined signal: the industry is moving from "AI does checkout" to "AI recommends, apps execute." The "agent app store" model—where agents route to specialized apps for specific tasks—is winning over the "one agent does everything" model.

### Technical Details

OpenAI checkout pivot:

| Metric | Instant Checkout | App-Based ACP | Change |
|--------|-----------------|---------------|--------|
| **Conversion Rate** | 1.9% | 4.8% (early data) | +153% |
| **User Trust** | 5.8/10 | 7.1/10 | +22% |
| **Average Order Value** | $43 | $72 | +67% |
| **Return Rate** | 18% | 9% | -50% |
| **Merchant Coverage** | 2.1M+ | 340K (growing) | -84% |

Wizard + Stripe ACP integration:

| Feature | Detail |
|---------|--------|
| **Payment Protocol** | ACP v1.2 native |
| **Merchant Network** | Stripe's 3.4M merchants |
| **Agent Framework** | Wizard agents (any model) |
| **Payment Methods** | Card, USDC, bank transfer |
| **Checkout UX** | Merchant's branded checkout (not Wizard's) |
| **Commission** | 0.5% to Wizard per transaction |

### Developer Integration Example

Wizard agent with ACP payment integration:

```python
# Wizard + Stripe ACP - Agent with native payments
from wizard import Agent, tool
from wizard.integrations import StripeACP

class ShoppingAgent(Agent):
    """An agent that can search and purchase products via ACP."""

    def __init__(self):
        super().__init__(
            name="shopping-assistant",
            model="claude-3.5-sonnet",
            # ACP integration - plug-and-play payments
            integrations=[
                StripeACP(
                    api_key=os.environ["STRIPE_KEY"],
                    # Let merchant handle checkout UX
                    checkout_mode="merchant-hosted",
                    # Agent commission
                    commission_rate=0.005  # 0.5%
                )
            ]
        )

    @tool
    async def search_products(self, query: str, max_price: float = None):
        """Search for products across ACP-integrated merchants."""
        results = await self.acp.search(
            query=query,
            filters={"maxPrice": max_price} if max_price else {},
            # Use UCP for discovery, ACP for payment capability
            protocols=["ucp", "acp"]
        )
        return results

    @tool
    async def purchase_product(self, product_id: str, quantity: int = 1):
        """Purchase a product using ACP - routes to merchant checkout."""
        # This creates a checkout session on the merchant's site
        checkout = await self.acp.create_checkout(
            product_id=product_id,
            quantity=quantity,
            # User completes payment in merchant's app/site
            redirect_url="https://merchant.com/checkout",
            # Agent metadata for attribution
            agent_metadata={
                "agent_id": self.agent_id,
                "platform": "wizard",
                "user_consent": "explicit"
            }
        )
        return {
            "checkout_url": checkout.url,
            "expires_at": checkout.expires_at,
            "total": checkout.total
        }

# Deploy the agent
agent = ShoppingAgent()
response = await agent.run(
    "Find me wireless earbuds under $100 with good noise canceling"
)
```

## Developer Takeaways

### Theme: The Walled Garden vs. Open Protocol War

The battle lines are drawn:

1. **Amazon's injunction** establishes that AI agents need consent to access merchant data—scraping is dead, protocols are mandatory
2. **Meta's Moltbook acquisition** shows Big Tech's strategy: absorb open marketplaces, add review gates, increase platform tax
3. **OpenAI's pivot + Wizard/Stripe** signals that the "agent app store" model (agents route to specialized apps) beats the "universal agent" model (one agent does everything)

### Implementation Priorities

1. **For Agent Developers**: Switch from web scraping to UCP/ACP for product data—the Amazon ruling makes scraping legally untenable
2. **For Moltbook Developers**: Begin migration to Meta's platform ASAP—the review process takes 2-5 days, and early movers get priority
3. **For Commerce Platforms**: Implement the "agent app store" pattern (Wizard/Stripe model)—agents recommend, your app handles checkout
